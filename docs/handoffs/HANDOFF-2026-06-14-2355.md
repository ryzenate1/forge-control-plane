## Important Findings

- Runtime coupling remains the highest-priority blocker.
- Cluster Manager still talks directly to daemon APIs.
- Docker-shaped resource semantics remain in node and capacity paths.
- Placement decisions are not durable and do not reserve capacity.
- Reconciler needs leader election before horizontal API scaling.
- Event Bus is useful as an internal foundation but needs versioning, correlation IDs, idempotency, and an outbox before NATS or other distributed messaging.
- Evacuation Planner correctly plans only, but plan status semantics should be clarified before migration execution.

## Risks

- Reconciler and Event Bus are currently process-local; multi-replica API deployments need lease/leader and durable event patterns first.
- Legacy compatibility fields can still drift if old paths mutate status fields directly.
