## Architecture Changes

- Event Bus envelopes now include `correlation_id`.
- Timeline entries are persisted from in-process events through an Observability Service subscriber.
- Timeline storage is idempotent by event ID to avoid duplicate rows for the same event.
- Server lifecycle, reconciliation, evacuation, and migration operations now carry correlation IDs.
- Heartbeat and health history are persisted separately from current node state.
- Operators can query timeline history globally, by resource, or by correlation ID.

## Known Limitations

- Event Bus remains in-process and is not distributed.
- Timeline persistence starts with API-process events only.
- Heartbeat expiry is not implemented.
- Recovery triggers are not implemented.
- Placement reservations are not implemented.
- Migration execution remains state-machine-only.
- Restore ownership remains daemon-coupled.
- No dashboards were added.
- No failover, recovery, runtime expansion, or AI features were implemented.
- Build, test, lint, typecheck, docker, and validation commands were intentionally not run.
