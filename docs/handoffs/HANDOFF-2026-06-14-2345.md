## Architecture Changes

- Nodes are now operator-managed lifecycle resources with active, maintenance, and draining desired states.
- Scheduler placement eligibility now depends on persisted node lifecycle state.
- Draining and maintenance block new workloads but do not affect existing workloads.
- Capacity exhaustion is tracked as both scheduler metrics and in-process events.
- Cluster visibility now includes region capacity and node lifecycle details.

## Known Limitations

- Draining does not migrate, stop, or fail over existing workloads.
- `NodeDrainingCompleted` currently means the node left draining state, not that workloads were evacuated.
- Health scoring is advisory and does not trigger automatic repair.
- Placement rejection metrics are in-memory and reset when the API process restarts.
- No failover, migration, runtime abstraction, NATS, external messaging, or AI features were implemented.
- Build, test, lint, typecheck, docker, and validation commands were intentionally not run.
- `gofmt` is not available in the current environment, so Go formatting was not automated.
