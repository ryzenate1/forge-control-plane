## Migration Notes

- Server state is backfilled from `servers.status`.
- Node desired state is backfilled from `maintenance_mode` and `draining`.
- Node actual state is backfilled from `nodes.status`.
- Legacy `status` remains in place and is synchronized for compatibility.
- The migration uses enum creation statements and is intended to run once through the existing `schema_migrations` runner.

## Known Limitations

- No Event Bus was implemented; transition records are database rows only.
- No failover, runtime abstraction, AI features, or auto migration were implemented.
- Some legacy workflow paths still use `status` for compatibility, but writes now also synchronize persisted state.
- The reconciliation loop still has no leader election for multi-API deployments.
- Build, test, lint, typecheck, docker, and validation commands were intentionally not run.
- `gofmt` is not available in the current environment, so Go formatting was not automated.
